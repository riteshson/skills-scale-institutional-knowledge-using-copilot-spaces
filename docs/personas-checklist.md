# Persona Checklist (Project-level)

Use this checklist to record which personas are assigned or engaged for a specific project. Add it to the project README or project board as needed.

- [ ] Project Manager (PM): ___________________
- [ ] Product Manager (PdM): ___________________
- [ ] Developers: ___________________
- [ ] Technical Lead / Architect: ___________________
- [ ] QA Lead / Test Engineer: ___________________
- [ ] DevOps / Platform Engineer: ___________________
- [ ] Release Manager: ___________________
- [ ] Security Engineer / AppSec Liaison: ___________________
- [ ] UX Researcher / Designer Liaison: ___________________
- [ ] Data Analyst / Insights Owner: ___________________
- [ ] Customer Success / Support Liaison: ___________________
- [ ] Compliance / Privacy Officer: ___________________

## Responsibility Mapping (example)

Artifact | Primary Owner | Secondary / Consulted
---|---:|---
PR description / acceptance criteria | Developer / PdM | QA, Security
Release notes / runbook | Release Manager | DevOps, PM
Risk Register entry | PM | Technical Lead, Security
Instrumentation / metrics | Data Analyst | PdM, Developers
Support Runbook | Customer Success | Developers, PM

## How to use
- Copy this checklist into the project README or onboarding doc.
- Update assignees at kickoff; review in the first weekly sync.
- Use the Responsibility Mapping to clarify who updates which artifacts.
